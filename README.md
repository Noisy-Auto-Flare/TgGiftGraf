# Telegram Gift Graph

Приложение для сбора, анализа и визуализации графа подарков в Telegram.

## Принцип работы краулера (Профессиональный обзор)

Краулер построен на базе **Telethon** и реализует стратегию "умного расширения" (Smart Expansion) графа:

1.  **Точки входа и сбор семян**:
    *   Краулер берет начальных пользователей из `START_USERNAMES`.
    *   Для массового пополнения базы он периодически сканирует публичные чаты/каналы из `TARGET_CHATS`, собирая участников и добавляя их в очередь.
2.  **Приоритизация и контроль очереди**:
    *   В базе данных `gifts.db` реализована очередь с приоритетами.
    *   Новые пользователи, найденные через связи (отправители подарков), обрабатываются в первую очередь.
    *   **Лимитирование**: Очередь ограничена параметром `MAX_CRAWL_QUEUE_SIZE`, что предотвращает бесконтрольное потребление ресурсов.
3.  **Инкрементальный сбор данных**:
    *   Краулер помечает пользователей временем `last_scanned`. Повторное сканирование происходит только через `RESCAN_THRESHOLD_DAYS`.
4.  **Имитация поведения человека (Anti-Flood)**:
    *   Добавлены **микро-паузы** между каждым API-вызовом (получение информации, поиск подарков, сканирование чатов).
    *   Используются случайные глобальные задержки между пользователями.
5.  **Надежность и обработка ошибок**:
    *   Краулер фильтрует ботов и удаленных пользователей.
    *   Обрабатываются специфические ошибки: `PrivateUserError`, `UserDeactivatedError`, `METHOD_NOT_AVAILABLE` (для Premium-фич).
    *   Логи записываются с ротацией (`RotatingFileHandler`), предотвращая переполнение диска.

## Режимы работы

1.  **Фоновый цикл (Daemon)**:
    Краулер работает постоянно, периодически сканируя чаты и обрабатывая очередь.
    ```bash
    python3 crawler.py
    ```
2.  **Режим однократного запуска (Cron)**:
    Если в `.env` установлено `CRAWL_SINGLE_RUN=true`, краулер обработает одного пользователя (или одну итерацию очереди) и выйдет. Идеально для запуска по расписанию:
    ```bash
    */10 * * * * cd /path/to/project && venv/bin/python crawler.py
    ```

## Файлы сессии и авторизация

Telethon использует файлы `.session` для хранения ключей авторизации.

- **Где хранить?** Файл сессии создается автоматически в корне проекта.
- **Первый запуск**: Запустите краулер вручную: `python3 crawler.py`. Введите номер телефона и код из Telegram.
- **Безопасность**: Файл сессии содержит доступ к аккаунту. Он добавлен в `.gitignore`.

## Установка и запуск

1.  **Подготовка**: `git clone`, `python3 -m venv venv`, `pip install -r requirements.txt`.
2.  **Конфигурация**: `cp .env.example .env`, заполните `API_ID`, `API_HASH`.
3.  **Инициализация БД**: `python3 database.py`.
4.  **Запуск API**: `uvicorn server:app --host 127.0.0.1 --port 8000`.
5.  **Запуск Аналитики**: `python3 analytics.py` (рекомендуется раз в несколько часов).

## Оптимизация для 1 ГБ ОЗУ
- **SQLite**: Минимум потребления памяти.
- **BFS & LPA**: Оптимизированные алгоритмы аналитики.
- **Caddy**: Легковесный веб-сервер с авто-HTTPS.
