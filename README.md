# Telegram Gift Graph

Приложение для сбора, анализа и визуализации графа подарков в Telegram.

## Принцип работы краулера (Профессиональный обзор)

Краулер построен на базе **Telethon** и реализует стратегию "умного расширения" (Smart Expansion) графа:

1.  **Точки входа и сбор семян**:
    *   Краулер берет начальных пользователей из `START_USERNAMES`.
    *   Для массового пополнения базы он периодически сканирует публичные чаты/каналы из `TARGET_CHATS`.
2.  **Приоритизация и контроль очереди**:
    *   В базе данных `gifts.db` реализована очередь с приоритетами.
    *   Новые пользователи, найденные через связи, обрабатываются в первую очередь.
    *   Очередь ограничена параметром `MAX_CRAWL_QUEUE_SIZE`.
3.  **Инкрементальный сбор данных**:
    *   Краулер помечает пользователей временем `last_scanned`. Повторное сканирование происходит только через `RESCAN_THRESHOLD_DAYS`.
4.  **Имитация поведения человека (Anti-Flood)**:
    *   Добавлены микро-паузы между каждым API-вызовом.
5.  **Надежность и обработка ошибок**:
    *   Краулер фильтрует ботов и удаленных пользователей.
    *   Логи записываются с ротацией.

## Развертывание через Docker Compose (Рекомендуется)

Самый быстрый и надежный способ запустить всё приложение на сервере.

### 1. Подготовка
```bash
git clone https://github.com/youruser/TgGiftGraf.git
cd TgGiftGraf
cp .env.example .env
# Отредактируйте .env (API_ID, API_HASH) и Caddyfile (ваш домен)
nano .env
nano Caddyfile
```

### 2. Первая авторизация (Важно)
Поскольку Telethon требует ввод кода из Telegram, первый запуск краулера нужно сделать интерактивно:
```bash
docker compose run --rm crawler python crawler.py
```
Введите номер телефона и код. После этого будет создан файл сессии, который сохранится в папке проекта.

### 3. Запуск всех сервисов
```bash
docker compose up -d
```
Эта команда запустит:
- **API**: Бэкенд на FastAPI.
- **Crawler**: Сборщик данных в фоновом режиме.
- **Analytics**: Периодический пересчет кластеров и охвата (раз в 6 часов).
- **Caddy**: Веб-сервер с автоматическим SSL и раздачей статики.

### 4. Управление
- Просмотр логов: `docker compose logs -f`
- Остановка: `docker compose down`

## Локальная установка (без Docker)

1.  **Подготовка**: `python3 -m venv venv`, `source venv/bin/activate`, `pip install -r requirements.txt`.
2.  **Инициализация БД**: `python3 database.py`.
3.  **Первая авторизация**: `python3 crawler.py`.
4.  **Запуск API**: `uvicorn server:app --host 127.0.0.1 --port 8000`.
5.  **Запуск Аналитики**: `python3 analytics.py`.

## Оптимизация для 1 ГБ ОЗУ
- **SQLite**: Минимум потребления памяти.
- **Docker**: Каждый контейнер ограничен по ресурсам и использует легкие образы.
- **Caddy**: Потребляет <50МБ ОЗУ.
